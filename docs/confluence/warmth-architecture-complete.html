1. Product overview

Warmth is a conference intelligence platform for founders, sales, and GTM teams. At a conference you might meet 50+ people; Warmth helps you:

  • Prepare — know who is worth meeting before you walk the floor
  • Capture — record and extract intelligence during the conversation, hands-free
  • Route — decide hot lead vs. community intro using warmth uplift
  • Follow up — draft context-rich emails (Lightfern polishes in Gmail; human sends)

Three surfaces, one backend

SurfaceRoleWhen
iOS + Apple WatchCapture — phrase trigger, speech, on-device NLPOn the conference floor
Web dashboardReview & manage — events, leads, warmth, follow-upsBefore/after the event
Python backendLifecycle pipelines, ML routing, integrationsAlways-on server

Surface split: capture happens on mobile; review/management on the web dashboard. Both talk to the same FastAPI backend + data layer.

5. iOS / watchOS architecture

App shell

RootView
 ├── OnboardingFlow (sign-in → permissions → calendar → finish)
 └── MainTabView
      ├── CaptureView      (RecordOrb, Waveform, wake word)
      ├── ConnectionsView  (session + seeded connections)

Composition root: AppModel

ServiceResponsibility
AuthProvidingFirebase Auth, ID tokens for signals
SpeechServicingAVAudioEngine → wake word → SFSpeechRecognizer
SocialGraphProcessingOn-device NLP → PersonNode
SignalSendingPOST CapturedSignal to backend + retry queue
SessionCaptureLogIn-memory session people (merge by name)
WatchSessionServiceiPhone ↔ Watch state sync
SettingsStoreOnboarding flag, base URL

Capture sequence

User says "hey it's nice to meet you" (or taps Record / Watch)
        │
        ▼
SpeechService.startRecording()
        │ live transcript (on-device)
        ▼
SocialGraphEngine.process(transcript)
        │ PersonNode: name, org, role, interests, icpScore, relations
        ▼
AppModel.capturePerson()
        ├── sessionLog.record(node)
        └── signalClient.send(CapturedSignal)  → POST /api/signals
        ▼
syncWatchState() → Watch complication updates

SocialGraphEngine (on-device NLP)

  • NER: NLTagger → PersonalName, OrganizationName
  • Relations: regex SPO — works_at, founded, leads, interested_in
  • ICP score: keyword proximity 0–100 (RevOps, GTM, SaaS, ML, etc.)
  • Output: PersonNode → encoded as CapturedSignal

Watch targets

  • WarmthWatch — remote start/stop capture
  • WarmthWatchWidget — complications showing recording state + last person
  • WatchConnectivity — bidirectional messages via WatchSessionService

8. ML pipeline & routing

Models (packages/ml/)

ModelInputOutput
WarmthModelICP fit (Zero) + enrichment (Unify) + engagement signalspredicted_score, actual_score, uplift, band
LeadScorerICP config + connection attrsIntent score; fallback ICP fit
LeadClustererMeetingSignal + community listcluster_id, nearest neighbors
MeetIntelligencePipelineMeetingSignal + prior WarmthScoreRoutingDecision (CRM vs community)

WarmthScore fields

  • icp_score — 0–100 from Zero CRM
  • warmth_score — 0–100 relational/intent warmth
  • predicted_score — pre-meet combined
  • actual_score — post-meet combined
  • uplift = actual − predicted → drives routing
  • band — cold / warm / hot

13. Deployment & infrastructure

Local development

# Backend
uv sync && uv run uvicorn apps.api.main:app --reload --port 8000

# Web
cd web && npm install && npm run dev   # http://localhost:5173

# iOS
cd iOS/Warmth-iOS && xcodegen generate
xcodebuild build -scheme Warmth -destination 'platform=iOS Simulator,name=iPhone 17 Pro'

Production target

  • API: GCP Cloud Run
  • DB: Firebase Firestore
  • Secrets: Google Secret Manager
  • Web: Static deploy (Vercel / Firebase Hosting)
  • iOS: TestFlight → App Store

14. Security

  • Firebase Auth + ID tokens on iOS signal payloads
  • Google Secret Manager for API keys (Deepgram, Zero, Unify, Tavily)
  • CORS restricted via WEB_ALLOWED_ORIGINS
  • On-device speech + NLP (privacy on capture)
  • Gmail drafts only — no auto-send without human action
